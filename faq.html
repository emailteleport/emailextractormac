Frequently Asked Questions

Find answers to common questions about EmailExtractorMax.

General

EmailExtractorMax is a native macOS application that helps you find and collect email addresses from multiple sources, including search engines (DuckDuckGo, Bing, Brave, Yahoo), local files, and email clients (Mac Mail, Outlook).
EmailExtractorMax supports macOS 11 (Big Sur) and later, including macOS 12 (Monterey), macOS 13 (Ventura), and macOS 14 (Sonoma). It runs natively on both Intel and Apple Silicon Macs.
EmailExtractorMax is a tool that extracts publicly available email addresses. It's legal to use for legitimate purposes like market research, lead generation, and contact discovery. However, you are responsible for using the collected data in compliance with applicable laws (CAN-SPAM, GDPR, etc.) and respecting websites' terms of service.

Features

EmailExtractorMax supports DuckDuckGo, Bing, Brave, and Yahoo search engines. You can search using a single engine or run parallel searches across all engines simultaneously for maximum coverage.
EmailExtractorMax can scan various file formats including: text files (.txt), Word documents (.doc, .docx), Excel spreadsheets (.xls, .xlsx), PDF files, HTML files, and email files (.emlx for Apple Mail). The app can recursively scan entire folders.
Deep crawling allows you to extract emails from not just search result pages, but also from the links found on those pages. You can configure the crawl depth (how many levels of links to follow) and concurrent task limits (how many pages to process simultaneously).
Yes! EmailExtractorMax includes powerful filtering options: blacklist specific domains, whitelist only certain patterns, exclude common spam addresses (noreply, info, etc.), and filter by email name patterns. These filters help you collect only the emails that matter.
You can export your extracted emails to TXT (one email per line) or CSV format. For CSV exports, you can customize the delimiter (comma, tab, or semicolon) and choose which columns to include: email address, source URL, search engine used, page title, and more.

Technical

Search engines may temporarily block requests if they detect automated access. EmailExtractorMax includes built-in rate limiting and user-agent rotation to minimize this. If you get blocked, try: reducing the number of concurrent tasks, increasing delays between requests, or waiting a few hours before resuming.
The Local Files and Email Client features work completely offline. However, the Search Engine feature requires an internet connection to query search engines and crawl web pages.
To improve speed: increase the concurrent task limit (but be careful not to trigger rate limiting), use the parallel multi-engine search feature, enable multi-threading for local file scanning, and use specific search queries to find more relevant results quickly.
